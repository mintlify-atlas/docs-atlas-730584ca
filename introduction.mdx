# Welcome to Kanagawa

Kanagawa is a high-level, imperative programming language for hardware design that introduces **Wavefront Threading** — a new execution model making concurrency explicit, enabling efficient and predictable parallel hardware.

Unlike traditional HLS tools that rely on pragmas and heuristics, Kanagawa lets you directly express concurrency, synchronization, and scheduling in code. This combines the productivity of software-like programming with the performance and efficiency of hand-written RTL.

## Why Kanagawa?

Kanagawa addresses a longstanding challenge in digital design: bridging the gap between high-level programming and efficient hardware implementation.

## Key Features

### Explicit Concurrency
Express parallelism directly in your code without relying on compiler heuristics. Wavefront Threading provides deterministic execution order while maintaining high performance.

### Competitive Performance
Kanagawa designs match or surpass hand-written RTL in area, performance, and power — often with less than half the code. Used in production for Microsoft Azure accelerators and cryptographic research.

### Composable Design
Build complex hardware from reusable components. The standard library includes data structures (arrays, FIFOs, hash tables), numeric types (fixed-point, floating-point, complex), and complete IP blocks like RISC-V processors.

### Hardware-Software Bridge
Write hardware designs using familiar programming constructs: functions, classes, loops, and conditionals. The compiler generates efficient SystemVerilog that integrates seamlessly with existing HDL workflows.

## Research Background

Kanagawa is based on research published in the ACM paper [Wavefront Threading Enables Effective High-Level Synthesis](https://dl.acm.org/doi/10.1145/3656420). The language and compiler have been validated in real-world applications including:

- Accelerators deployed in Microsoft Azure
- Cryptographic research prototypes
- Full RISC-V processor cores with multi-hart support
- Custom instruction extensions and specialized compute engines
