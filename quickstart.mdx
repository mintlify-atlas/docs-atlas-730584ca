## What is Kanagawa?

Kanagawa is a high-level, imperative programming language for hardware design that introduces **Wavefront Threading**, a new execution model that makes concurrency explicit, enabling efficient and predictable parallel hardware.

Unlike traditional HLS tools, which often require pragmas and heuristics, Kanagawa allows you to directly express concurrency, synchronization, and scheduling in code. This combines the productivity of software-like programming with the performance and efficiency of hand-written RTL.

## Your First Kanagawa Program

Here's a simple example that demonstrates Kanagawa's syntax and threading model:

```kanagawa
class Counter
{
private:
    uint32 _count = 0;

public:
    uint32 Increment(uint32 value)
    {
        uint32 result;
        
        atomic
        {
            result = _count;
            _count += value;
        }
        
        return result;
    }
}

export Counter;
```

This example shows:
- **Class-based organization**: Hardware is structured using classes
- **Typed integers**: `uint32` for 32-bit unsigned values
- **Atomic blocks**: Ensures thread-safe access to shared state
- **Export declarations**: Makes the class available as a Verilog module

## Understanding Threads

In Kanagawa, a thread is a runtime construct defined by:
- **Context**: Values for each local variable
- **Location**: The next statement to execute

Threads physically progress through generated hardware pipelines:

```kanagawa
class Processor
{
    uint32 Process(uint32 x)
    {
        uint32 y = x + 2;      // Stage 1
        uint32 z = y * 10;     // Stage 2
        return z / 4;          // Stage 3
    }
}
```

This generates a 3-stage pipeline where multiple threads can be in-flight simultaneously.

## Quick Reference

### Integer Types

```kanagawa
uint7 x;   // Unsigned 7-bit integer
int66 y;   // Signed 66-bit integer
auto z = x + y;  // Type inference
```

### Control Flow

```kanagawa
// Static unrolled loop
static for (const auto i : 8)
{
    result[i] = input[i] * 2;
}

// Pipelined loop with dynamic thread count
pipelined_for(count, [](uint32 tid)
{
    // Loop body
});
```

### Synchronization

```kanagawa
// Atomic block for thread-safe operations
atomic
{
    static uint32 counter = 0;
    counter++;
}

// Wait until condition is met
atomic do {} while (ready_flag == false);
```

<Note>
Kanagawa designs are highly competitive with hand-written RTL in area, performance, and power, while requiring far less code.
</Note>
